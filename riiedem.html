RIIEDEM

Repositorio Institucional Para Instituciones Educativas Del Estado De México

En este proyecto, trabajé como miembro del equipo en la implementación de la propuesta, el desarrollo del diseño del sistema, el desarrollo del sistema, la implementación de una aplicación nativa y su despliegue.

Este proyecto trata sobre la creación de un repositorio institucional para diversos planteles educativos. El objetivo principal es preservar la información de los docentes, permitiéndoles almacenar datos relacionados con su información académica.

El propósito del proyecto es proponer un repositorio institucional que facilite los procesos administrativos de los docentes. Para lograr esto, el sistema cuenta con dos tipos de perfiles: uno para los docentes y otro para los administradores.

Para las sesiones de los docentes, el sistema incluye apartados para almacenar información como: detalles de sus clases, participación en congresos, estancias académicas de los alumnos, horarios de clases, informes de estadías, información sobre libros o capítulos publicados, modelos de utilidad, patentes y publicaciones. También cuenta con un apartado para visualizar su información personal.

En cuanto al perfil del administrador, tiene funciones como dar de alta usuarios, actualizar su información, eliminarlos y visualizarlos. Además, el sistema cuenta con un apartado de registro de usuarios para que cada uno pueda darse de alta, y también ofrece la opción de recuperar la contraseña en caso de extravío.

Mi responsabilidad principal en este proyecto, como mencioné, fue:
-Elaborar encuestas para conocer las necesidades del cliente y las opiniones de los posibles usuarios.
-Plantear la solución a esas necesidades, determinando que se trataría de un sitio web con una aplicación para dispositivos móviles.
-Desarrollar el diseño de las interfaces del sistema.
-Desarrollar los formularios de registro, inicio de sesión y recuperación de contraseña.
-Implementar los formularios para el almacenamiento de la información de las clases, congresos de participación, estancias académicas, horarios, informes de estadía, información sobre libros o capítulos, modelos de utilidad, patentes y publicaciones.
-Permitir a cada docente registrar, modificar, consultar y eliminar su información en cada una de las áreas mencionadas anteriormente.
-Implementar el apartado de bienvenida una vez iniciada la sesión de los docentes.
-Implementar el apartado de perfil que contiene la información con la que el docente se registró.
-Desarrollar la base de datos para almacenar la información de cada uno de los apartados antes mencionados.
-Diseñar, desarrollar e implementar el software como una aplicación nativa para su funcionamiento en dispositivos móviles.
-Implementación de pruebas de integridad, funcionalidad y de humo.

En cuanto a la propuesta del repositorio institucional como solución al problema de facilitar los procesos administrativos, se propuso desarrollarlo utilizando HTML, CSS, JavaScript, Bootstrap, PHP y MySQL.

En relación a la elaboración de la encuesta para el cliente, se plantearon las siguientes preguntas:

1. ¿A quién va a estar dirigido el repositorio insitucional?
2. ¿Cuál es el propósito de este repositorio?
3. ¿Qué diferencia tendrá este repositorio de entre el resto de repositorios existentes con la misma función o una función parecida?
4. ¿Tiene algún requerimiento en específico para el repositorio institucional?
5. ¿Cuenta con alguna idea sobre cómo quiere que se vea? De preferencia coloque algún ejemplo.
6. ¿Ha considerado algún color o esquema definido para el repositorio?
7. ¿Hay alguna funcionalidad o diseño que le gustaría implementar de otros sitios web?
8. ¿Hay algún contenido qué le gustaría que se brindará a través de este repositorio? (Fotos, vídeos, imágenes, archivos PDF, etc.).
9. ¿Requiere que tenga algún apartado de contacto?
10. ¿Ha planeado implementarlo en alguna institución?
11. ¿Qué evidencias requiere para la entrega del repositorio?

Y en cuanto a la elaboración de la encuesta para los posibles usuarios, se formularon las siguientes preguntas:
1. Edad.
2. ¿Qué tan importante es para usted que exista un repositorio institucional?
3. ¿Es fundamental para usted que se puedan almacenar sus horarios actuales y pasados además de poder almacenar el nombre de sus clases impartidas y clases especiales?
4. ¿Qué tan valioso es para usted poder almacenar certificados de publicaciones?
5. ¿Para usted qué tan crucial es poder almacenar sus constancia de partición en congresos?
6. ¿Qué tan provechoso es para usted poder almacenar sus patentes?
7. ¿Si almacenará sus modelos de utilidad que tan significativo sería?
8. ¿Qué tan esencial es para usted el poder almacenar sus libros o capítulos de libros?
9. ¿Poder almacenar los reportes de estancias académicas e informes técnicos qué tan valioso es para usted?
10. ¿Qué tan conveniente es para usted poder almacenar distintos formatos para su posterior descarga?

Este proyecto fue realizado en base a la metodología Scrum, siguiendo las fechas indicadas de los sprints para su posterior testeo y despliegue. Se aplicaron pruebas de integración, funcionalidad y humo.

De esta forma, se puede establecer que con las pruebas de integración aplicadas al sistema de RIIEDEM, cada apartado del sistema es funcional y que cualquier enlace, botón u opción disponible, devolverá una respuesta de la acción antes realizada. Por ejemplo, al posicionarse sobre el menú, este despliega un listado de todas las opciones posibles y al dar clic en cualquiera de ellas, se verá reflejada una acción.

Con esto, las pruebas funcionales sirvieron para verificar que las acciones realizadas devuelvan un valor esperado. Es decir, si se decide ingresar el registro de una nueva clase, para posteriormente poder visualizar los datos de esta, se puede insertar la información deseada y, en efecto, se devuelvan los valores antes ingresados.

Por último, con las pruebas de humo, se pudo establecer que la funcionalidad básica del sistema es operable sin problemas. El formulario de registro de usuarios funciona sin problema, los inicios de sesión validan correctamente la información y cada uno de los formularios para subir información nueva de las clases, horarios, patentes, etc., además de actualizarla, eliminarla o visualizarla, funcionan sin problemas, devolviendo los valores esperados. Además, se puede realizar una descarga del repositorio y una instalación automática del mismo en el equipo de cómputo o el dispositivo móvil.
